# EMS Wireframe Plan

Target fidelity: low-to-mid fidelity for product alignment before high-fidelity UI.

## Priority Screens

1. **Auth**
   - Sign in
   - Forgot password
   - MFA challenge

2. **Dashboard**
   - Global metrics
   - Upcoming events list
   - Alerts panel

3. **Event Management**
   - Event list with filters
   - Event detail
   - Create/Edit event form

4. **Venue & Room Management**
   - Venue list
   - Room capacity/availability view

5. **Attendee Management**
   - Attendee list + search
   - Check-in panel

6. **Audit & Admin**
   - Audit log timeline/table
   - Role & permission management

## Wireframe Conventions
- 12-column desktop grid; 4-column mobile grid.
- Annotate key interactions (filtering, status changes, save actions).
- Mark responsive adaptation notes per screen.
- Include empty, loading, and error states.

## Handoff Checklist
- Include page objective and user role.
- Identify primary CTA and secondary actions.
- Map each block to design-system components.
- Note analytics events for major interactions.
